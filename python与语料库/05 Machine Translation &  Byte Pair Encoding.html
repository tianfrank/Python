机器翻译¶

_（部分内容参考PyTorch Tutorial实现）_

在这个项目中我们将实现一个简单的基于循环神经网络的机器翻译系统，用于英文到中文的翻译。当然，由于我们会使用玩具数据集(toy dataset)，翻译的效果是没有保证的。即便如此，我们会看到基于神经网络的模型强大的表达能力。需要注意的是，即便是现今最先进的机器翻译系统，在公开评测中还是无法超越人类的。比如，在WMT 2019的评测中，仅有Facebook-FAIR模型在英德翻译上取得了超越人类的效果，大多数模型在英德互译效果仅与人相当。[1]

回顾：在机器翻译中，序列到序列模型，首先将源语言编码为向量表示，然后根据该向量表示解码出目标语言的概率分布。解码过程中会利用注意力机制(attention)不断补充源语言的信息。最终句子的生成，可以由概率分布通过贪心算法(greedy search)得到，即每次只选择最可能的词作为输出；也可以通过柱搜索算法(beam search)得到，即每次保留最有可能的前几个子序列。

通过序列到序列(sequence-to-sequence)的模型框架，一个基础的机器翻译系统可以被简洁的实现，从代码的编写的角度可以被分为以下的构件：

  • 数据处理
  • 模型定义
  • 训练循环
  • 交互测试

数据导入¶

我们使用的玩具数据中包含几万条平行中英句子对。我们需要将数据转换为计算机易于计算的形式，首先我们将对句子进行简单的清理，然后进行分词，最后将字符串转换为数字形式，最后一步的转换需要一个预先定义好的“词典”。

由于语言中的词数量是非常庞大的，实际中我们会控制词的数量在固定的范围内，常见的做法是根据训练集中的词频筛掉频率较低的词。原因一个是计算机的资源受限，另一个是对于统计机器学习系统来说低频数据无法提供足够可靠的统计信息，引入的噪音反而会加重过拟合的问题。

def clean_latin_text(s):
    # 按空格分词
    s = s.lower().strip() # 小写并清除首尾空白字符
    s = re.sub(r"([.!?])", r" \1", s) # 标点前添加空格
    s = re.sub(r"[^a-zA-Z.!?]+", r" ", s) # 清除其它符号
    return s

def clean_zh_text(s):
    # 分字
    s = s.strip()
    s = ' '.join([char for char in s])
    return s


def filter_by_en(en, zh):
    # 只保留简单的句子
    return (len(en.split(' ')) < MAX_LENGTH and
        len(zh.split(' ')) < MAX_LENGTH and
        en.startswith(PREFIXS))

class Vocab:
    def __init__(self, name):
        self.name = name
        self.word2index = {'<SOS>': 0, '<EOS>': 1}
        self.word2count = {}
        self.index2word = {0: '<SOS>', 1: '<EOS>'}


    def __len__(self):
        return len(self.word2index)

    def add_sentence(self, sentence):

        for word in sentence.split(' '):
            self.add_word(word)

    def add_word(self, word):
        if word not in self.word2index:
            self.word2index[word] = len(self)
            self.word2count[word] = 1
            self.index2word[len(self)-1] = word
        else:
            self.word2count[word] += 1

模型架构¶

torch中的模型定义需要采用面向对象的基于继承的方式(继承torch.nn.Module类)完成。模型由编码器、解码器组成。我们在解码器中一并实现了基于全连接层的注意力机制。注意力的计算有很多种实现方式，但目标都是对两个向量计算一个分数。

一般而言，为了充分利用计算资源，模型计算会以批量的形式进行，即一次处理多个句子。但一次多个句子的处理时，需要额外考虑句长不同的问题。为了实现上的简易性，我们这里每次只处理一个句子。

class EncoderRNN(nn.Module):
    def __init__(self, input_size, hidden_size):
        super(EncoderRNN, self).__init__() # 必须在添加参数前初始化基类
        self.hidden_size = hidden_size

        self.embedding = nn.Embedding(input_size, hidden_size)
        self.gru = nn.GRU(hidden_size, hidden_size, batch_first=True)

    def forward(self, input):
        # 输入单位为句子
        embedded = self.embedding(input)
        output, hidden = self.gru(embedded)
        return output, hidden


class AttnDecoderRNN(nn.Module):
    def __init__(self, hidden_size, output_size):
        super(AttnDecoderRNN, self).__init__()
        self.hidden_size = hidden_size
        self.output_size = output_size

        self.embedding = nn.Embedding(self.output_size, self.hidden_size)
        self.gru = nn.GRU(self.hidden_size, self.hidden_size)
        self.attn = nn.Linear(self.hidden_size * 2, 1)
        self.attn_combine = nn.Linear(self.hidden_size * 2, self.hidden_size)
        self.out = nn.Linear(self.hidden_size, self.output_size)

    def forward(self, input, hidden, encoder_outputs):
        # 输入单位为词
        # hidden should be [1, 1, hidden]
        # encoder_outputs shoud be [1, src_len, hidden]
        embedded = self.embedding(input).view(1, 1, -1)

        # [1, 1, hidden], [1, 1, hidden]
        output, hidden = self.gru(embedded, hidden)

        # [1, src_len]
        # 这里Attention的实现方法是，将解码器的隐状态与编码器的输出合并
        # 然后经过一个变化，输出每个位置对的分数
        attn_weights = self.attn(torch.cat([hidden.expand_as(encoder_outputs), encoder_outputs], dim=-1)).view(1, -1)
        attn_weights = F.softmax(attn_weights, dim=1)

        # [1, hidden] 这里是对编码器输出进行加权平均
        attn_applied = torch.mm(attn_weights, encoder_outputs[0])

        # [1, hidden]
        output = F.relu(self.attn_combine(torch.cat([output[0], attn_applied], dim=1)))

        output = self.out(output)

        return output, hidden, attn_weights

def binarize_text(line, vocab):
    idxs =  [vocab.word2index[word] for word in line.split(' ')]
    return torch.tensor(idxs, dtype=torch.long, device=device).view(1, -1)

def string_text(tensor, vocab):
    idxs = tensor.tolist()[0]
    return ' '.join([vocab.index2word[idx] for idx in idxs])

def tensor_from_pair(pair):
    return binarize_text(pair[0] + ' <EOS>', vocab_en), binarize_text('<SOS> ' + pair[1] + ' <EOS>', vocab_zh)

def recover_from_tensor(pair):
    return string_text(pair[0], vocab_en), string_text(pair[1], vocab_zh)

# 每个训练步骤，由前向传播、反向传播和参数更新构成
def train_step(input_tensor, target_tensor,
          encoder, decoder,
          encoder_optimizer, decoder_optimizer,
          criterion):

    # optimizer
    encoder_optimizer.zero_grad()
    decoder_optimizer.zero_grad()

    input_length = input_tensor.size(1)
    target_length = target_tensor.size(1)

    # encode [1, src_len, hidden_size], [1, 1, hidden_size]
    encoder_outputs, encoder_hidden = encoder(input_tensor)

    # decode
    decoder_outputs = []
    decoder_hidden = encoder_hidden # [1, 1, hidden_size]

    # Teacher forcing
    for di in range(target_length - 1):
        # [1, out], [1, 1, hidden], [1, src_len]
        decoder_output, decoder_hidden, decoder_attention = decoder(
            target_tensor[:, di], decoder_hidden, encoder_outputs)
        decoder_outputs.append(decoder_output)

    decoder_outputs = torch.stack(decoder_outputs, dim=1)[0] # [tgt_len, vocab]
    decoder_targets = target_tensor[0, 1:] # [tgt_len]
    loss = criterion(decoder_outputs, decoder_targets)
    # backprop
    loss.backward()

    # update
    encoder_optimizer.step()
    decoder_optimizer.step()

    return loss.item() / (target_length - 1)

# 整个训练的循环，我们训练固定的次数
def train(encoder, decoder, n_iters, print_every=100,
               learning_rate=0.001):
    reset_seed()
    print_loss_total = 0  # Reset every print_every

    # 初始化优化器和优化目标
    encoder_optimizer = optim.Adam(encoder.parameters(), lr=learning_rate)
    decoder_optimizer = optim.Adam(decoder.parameters(), lr=learning_rate)
    criterion = nn.CrossEntropyLoss()

    for iter in range(1, n_iters + 1):
        training_pair = tensor_from_pair(random.choice(pairs)) # 随机选择一个句子对进行训练
        input_tensor = training_pair[0]
        target_tensor = training_pair[1]

        loss = train_step(input_tensor, target_tensor, encoder,
                     decoder, encoder_optimizer, decoder_optimizer, criterion)
        print_loss_total += loss

        if iter % print_every == 0:
            print_loss_avg = print_loss_total / print_every
            print_loss_total = 0
            print(f'| iter {iter:03d}/{n_iters:03d} | loss {print_loss_avg:.4f}')

def inference(encoder, decoder, sentence):
    with torch.no_grad():
        input_tensor = binarize_text(clean_latin_text(sentence) + ' <EOS>', vocab_en)
        input_length = input_tensor.size()[1]

        # encode [1, src_len, hidden_size], [1, 1, hidden_size]
        encoder_outputs, encoder_hidden = encoder(input_tensor)

        # decode
        decoder_words = ['<SOS>']
        decoder_hidden = encoder_hidden # [1, 1, hidden_size]
        decoder_attentions = []

        for _ in range(MAX_LENGTH):
            last_word = decoder_words[-1]
            last_word_idx = vocab_zh.word2index[last_word]
            decoder_input = torch.tensor([[last_word_idx]], device=device)

            # [1, out], [1, 1, hidden], [1, src_len]
            decoder_output, decoder_hidden, decoder_attention = decoder(
                decoder_input, decoder_hidden, encoder_outputs)
            decoder_attentions.append(decoder_attention)

            word_idx = torch.argmax(decoder_output[0])
            word = vocab_zh.index2word[word_idx.item()]
            decoder_words.append(word)
            if word == '<EOS>':
                break

        # [tgt_len, src_len]
        decoder_attentions = torch.cat(decoder_attentions, dim=0).tolist()

        return decoder_words, decoder_attentions

BPE¶

Byte Pair Encoding，顾名思义是根据Byte Pair对进行编码，原本是一种通用的数据编码与压缩方法。

Byte pair encoding or digram coding is a simple form of data compression in which the most common pair of consecutive bytes of data is replaced with a byte that does not occur within that data. A table of the replacements is required to rebuild the original data. The algorithm was first described publicly by Philip Gage in a February 1994 article "A New Algorithm for Data Compression" in the C Users Journal.

在NLP的应用中，BPE技术得到进一步扩展，从字符序列开始，每次基于一对字符串编码文本，即将最常出现的相邻的字符串拼合为新的字符串，循环往复直到结束。

编码后的文本是被称为subword unit的序列。subword unit是介于字符与词之间的一种单元。类似语言学中word与morpheme的级别关系：词素是构成词的要素，是语言中最小单位的音义结合体；而词是语言中能够独立运用的最小单位。但subword unit与的最大的区别是subword unit是基于数据统计信息划分的，不一定具有原子性的含义。比如，“葡萄”这个短语既是词语也是词素，但BPE编码有可能将其编码为“葡”和“萄”。

BPE可以有效的解决按词进行文本处理导致的未登录词(out-of-vocabulary, OOV)问题。现有的算法都依赖于固定大小的词汇表来处理文本。基于词为单位进行编码虽然逻辑上最为合理，但在实践中有很多问题。比如词表爆炸的问题，根据Zipf定律，语言中大部分的词是低频词，如果把只出现一两次的词都算到词表中，词表会巨大无比，而且基于统计的模型对于低频词是捕获不到多少信息的。未登录词是指那些没有在训练语言中出现过的词，算法根本就无法获得其统计信息的，但真实文本中又往往经常出现。现在解决OOV有很多方法，比如基于字符/字编码和字符词汇联合编码等。BPE通过一个有限的词表，可以编码所有的文本，同时保留了一定词语结构上的信息。

基于频率的BPE算法与Python简易实现¶

Sennrich et al. (2016)于ACL 2016年提出了使用BPE在机器翻译中的构建subword词表。BPE的应用分为两个过程，首先是学习BPE的组合，其次是在新文本上应用BPE获得subword的序列。

学习算法流程如下：

  1. 准备一个足够大的训练数据，用于学习BPE
  2. 定义一个理想的subword词典大小
  3. 将“词”切成字符片段，并在每个词后面附加元符号“</w>”表示词结尾，然后补充词的频率，例如“low”的频率为5，我们将记录这样一条记录
    "l o w </w>": 5
  4. 选择最高频率的“基本单元”对组成一个新的subword，并记录一个“组合”
  5. 重复第4步，直到达到预定义的词典大小，或者下一个最高频率对的频率为1

def get_stats(vocab):
    """Compute frequencies of adjacent pairs of symbols."""
    pairs = collections.defaultdict(int)
    for word, freq in vocab.items():
        symbols = word.split()
        for i in range(len(symbols)-1):
            pairs[symbols[i],symbols[i+1]] += freq
    return pairs

def merge_vocab(pair, v_in):
    v_out = {}
    bigram = re.escape(' '.join(pair))
    p = re.compile(r'(?<!\S)' + bigram + r'(?!\S)')
    for word in v_in:
        w_out = p.sub(''.join(pair), word)
        v_out[w_out] = v_in[word]
    return v_out

  1. 将“词”切成字符片段，并在每个词后面附加元符号“</w>”表示词结尾
  2. 查找在“组合”表中频率最高的相邻单元对，并合并
  3. 重复2直到只剩一个基本单元或者剩下的单元对都不在组合表中

def get_pairs(word):
    """Return set of symbol pairs in a word.
    Word is represented as a tuple of symbols (symbols being variable-length strings).
    """
    pairs = set()
    prev_char = word[0]
    for char in word[1:]:
        pairs.add((prev_char, char))
        prev_char = char
    return pairs


def encode(orig):
    """Encode word based on list of BPE merge operations, which are applied consecutively"""

    word = tuple(orig) + ('</w>',)
    print("word split into characters: {}".format(word))

    pairs = get_pairs(word)

    if not pairs:
        return orig

    iteration = 0
    while True:
        iteration += 1
        show_bold(f"Iteration {iteration}:")

        print(f"bigrams in the word: {pairs}")
        bigram = min(pairs, key = lambda pair: bpe_codes.get(pair, float('inf')))
        print(f"candidate for merging: {bigram}")
        if bigram not in bpe_codes:
            print("Candidate not in BPE merges, algorithm stops.")
            break
        first, second = bigram
        new_word = []
        i = 0
        while i < len(word):
            try:
                j = word.index(first, i)
                new_word.extend(word[i:j])
                i = j
            except:
                new_word.extend(word[i:])
                break

            if word[i] == first and i < len(word)-1 and word[i+1] == second:
                new_word.append(first+second)
                i += 2
            else:
                new_word.append(word[i])
                i += 1
        new_word = tuple(new_word)
        word = new_word
        print(f"word after merging: {word}")
        if len(word) == 1:
            break
        else:
            pairs = get_pairs(word)

    # don't print end-of-word symbols
    if word[-1] == '</w>':
        word = word[:-1]
    elif word[-1].endswith('</w>'):
        word = word[:-1] + (word[-1].replace('</w>',''),)

    return word

Sennrich et al. (2016)提出的BPE是目前翻译中应用最为广泛的subword拆分方式。NMT模型擅长翻译文本的流畅性，但在未登录词上精度较SMT模型弱不少，特别是中文（基于词的模型）、屈折形式复杂的语言（德语、西班牙语）。通过BPE，可以大幅降低未登录词的数量，提升NMT模型的可用性。在一些语言中甚至可以将未登录词数量降为0，比如英语。但对于中文来说，还是会有少量未登录词。一个现有的研究表明，在中英翻译中，中文词个数有20多万，使用常用的3万大小的词表，会出现17万的未登录词；而使用3万大小的BPE词表，可以将未登录词数量将为5百多；而英语10万词可以用不到3万的BPE完全表示。

Sennrich et al. (2016)开源了代码，项目名为subword-nmt。同时，也有其它的项目复现了该算法，比如fastBPE等。该BPE方法也被用于GPT、RoBERTa等预训练模型。在很多任务上，是否使用BPE在模型结果上有天壤之别。有很大的可能，在基线模型上使用BPE可以超过一系列基线模型的改进。

基于语言模型的BPE¶

WordPiece¶

WordPiece是另一种切分方法，于BPE类似。Schuster and Nakajima在2012年提出，来解决日语和汉语的语音识别问题。

流程如下：

  1. 准备一个足够大的训练数据
  2. 定义一个理想的subword词典大小
  3. 将“词”切成字符片段
  4. 根据3的结果学习一个语言模型
  5. 选择cib所有可能的新词中能使训练数据似然增高最大的那个词，加入subword词典，并记录一个“组合”
  6. 重复第5步，直到达到预定义的词典大小，或者似然增高低于某个阈值

Unigram Language Model¶

Kudo (2018)于ACL 2018提出的另一种方法，借助Unigram Language Model，基于subword出现独立的假设。主要使为了解决subword切分歧义的问题。比如“bookkeeper”这个词可能被拆为“book keeper”或者“book keep er”，BPE采用贪心的方式，每次组合最高频率的单元对，但组合结果不一定是最合理的。Kudo引入Unigram Language Model对组合进行正则化，期望获得困惑度最小的subword序列。流程不再赘述。

BPE有很广泛的应用，在词向量学习中(fasttext)、语言理解(BERT等)、语言生成(GPT等)，是一项实用、有效的技术。

Transformers¶

transformers工具包提供的预训练模型中，BERT使用的Tokenzier是基于WordPiece的，GPT、XLM、XLNet、RoBERTa是基于BPE的，TransformerXL没有使用subword切分。我们可以通过调用工具包提供的方法来复用已经训练好的subword切分方式。需要注意的是BPE方法一般均需要提前进行基本的tokenization。transformers中的Tokenizer首先会执行特定的tokenization、然后应用BPE，同时不同模型BPE的训练语料也不完全相同，因而即便使用了同一种BPE算法，对一个句子的编码很可能不一致。比如：

  • BERT使用的tokenization仅处理标点
  • GPT使用的tokenization是首先ftfy正规化文本、然后使用spaCy的tokenizer
  • XLM使用的tokenization是机器翻译工具包MOSES提供的

def transformers_tokenize_sentence(sentence):
    print(f'sentence:\n{sentence}')
    print()
    for name, tok in tokenizers.items():
        print(f'{name}:\n{tok.tokenize(sentence)}')
        print()
        print(f'{name} (recover):\n{tok.decode(tok.encode(sentence))}')

以上的切分结果中，反映出了几个需要主要的事项：

  • 注意是否需要预先tokenize
  • subword切分方式是与其训练数据密切相关的，不同的数据会导致不同的划分结果
  • subword要注意应用到的语言与训练语言是否一致，否则仍然会出现非常多的UNK，OOV依然严重
  • 分词后的结果，有很多事无法还原成原文本的
  • subword中一些诡异的切分，比如
    • bert-base-cased将'unexplored'切分为'une x p lore d'
    • xlm-mlm-tlm-xnli15-1024将'explored'切分为'explo red', 'previously'切分为'previ ously', 'researchers'切分为'rese arch ers'
      引入的噪音，会提高模型过拟合的风险，也暴露出现有BPE算法基于频率贪婪合并相连单元的问题。 一种替代的目标是选择可以使最终切分片段最少的合并，但需要动态规划技术大幅增加了计算复杂度。
